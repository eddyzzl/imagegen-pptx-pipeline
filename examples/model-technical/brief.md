# Model Technical Deck Example

Use `$imagegen-pptx-pipeline` to create a 12-slide model technical review deck.

## Brief

We need to present a new risk scoring model to a technical governance committee. The deck should cover business objective, data scope, feature design, model architecture, evaluation results, stability checks, explainability, monitoring plan, limitations, and rollout recommendation.

## Constraints

- Deck profile should be `model-technical`.
- Visual proof objects should include architecture, data lineage, evaluation scorecards, monitoring loops, and rollout controls.
- Do not invent metrics. Ask for missing data or label assumptions.
- Produce multiple distinct styles, such as technical schematic, glass dashboard, editorial data story, and system blueprint.
- Rebuild the final PPTX from approved ImageGen comps without collapsing diagrams into plain tables.
